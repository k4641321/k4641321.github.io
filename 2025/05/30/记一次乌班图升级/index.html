记一次乌班图升级

2025/5/30 系统&软件 乌班图 Linux

起因

因为运行开源项目时，需要nodejs，但是我当前版本的不支持，以及后面python的要求，被迫升级
在升级的途中，我是用ssh连过去升级的，但因为网络不稳定，升级途中网络断了，连上守护进程勉强升到了24LTS，但后面迎来的是网络配置错误

遇到的问题

使用ping时，ping ip会提示：ping network is unreachable，ping域名又会显示：temporary failure resolving，apt update会显示temporary failure resolving，按网上讲的，就是网络配置有问题，但是！
按网上讲的用netplan设置，但我刚升级的乌班图没有，许多方法也是无用功，最后找到了这一条 

sudo ifconfig 网络适配器名称 192.168.0.101 netmask 255.255.255.0  //配置ip地址 和 子网掩码

接着 

sudo route add default gw 192.168.0.1  //设置网关

这样就把网络配置好了，可以正常使用了
